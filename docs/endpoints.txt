DOKUMENTACJA API - WYSYŁANIE ZDJĘĆ (POSIŁKI I ETYKIETY)

Adres serwera (Base URL): http://192.168.0.41/api/

WAŻNE DLA ANDROIDA: Serwer działa po HTTP. Pamiętajcie o dodaniu android:usesCleartextTraffic="true" w pliku AndroidManifest.xml, inaczej apka nie połączy się z serwerem.

WAŻNE (AUTORYZACJA): Wszystkie poniższe endpointy wymagają wysłania w nagłówku tokenu JWT zalogowanego użytkownika!
Nagłówek: Authorization: Bearer <wasz_token>
(Nie wysyłamy już user_id w JSON-ie, backend sam wie, kto to jest na podstawie tokenu).

KROK 1: POBRANIE LINKU (Presign)
Aplikacja prosi o wygenerowanie linku do wysyłki. Dla posiłków musicie wykonać ten krok 3 razy (dla przodu, lewej i prawej strony).

Metoda: POST
Endpoint: photos/presign
Co wysłać (JSON):
{
"extension": "jpg",
"folder": "meals" // Zmieńcie na "labels", jeśli to zdjęcie etykiety
}

Co otrzymacie (Odpowiedź):
{
"photo_id": "550e8400-...",
"object_key": "meals/user_1/550e8400-....jpg",
"upload_url": "http://192.168.0.41/..."
}

Zapiszcie sobie photo_id oraz upload_url!

KROK 2: FIZYCZNY UPLOAD PLIKU
Wysłanie bajtów zdjęcia pod adres otrzymany w Kroku 1.

Metoda: PUT
Adres URL: Użyjcie pełnego linku z pola "upload_url" (z Kroku 1).
Nagłówki (Headers): Content-Type: image/jpeg
Body: Czysty plik (binary raw bytes). To NIE jest multipart/form-data.

KROK 3: POTWIERDZENIE (Zapisanie w bazie)
Informacja dla backendu, że pliki zostały wysłane. Wybierzcie odpowiedni endpoint w zależności od tego, co wysyłacie.

OPCJA A: Zapis posiłku (Wymaga 3 zdjęć)
Metoda: POST
Endpoint: meals
Co wysłać (JSON):
{
"front": { "photo_id": "<id-z-kroku-1>", "width": 1920, "height": 1080, "extension": "jpg" },
"left": { "photo_id": "<id-z-kroku-1>", "width": 1920, "height": 1080, "extension": "jpg" },
"right": { "photo_id": "<id-z-kroku-1>", "width": 1920, "height": 1080, "extension": "jpg" }
}

Nowość: odpowiedź z endpointu meals zwraca dodatkowo pole:
"classification_status": "pending"

To oznacza, że backend rozpoczął klasyfikację zdjęć jedzenia w tle.

OPCJA B: Zapis etykiety (Wymaga 1 zdjęcia)
Metoda: POST
Endpoint: labels
Co wysłać (JSON):
{
"photo": { "photo_id": "<id-z-kroku-1>", "width": 1920, "height": 1080, "extension": "jpg" }
}

Nowość: odpowiedź z endpointu labels zwraca dodatkowo pole:
"classification_status": "not_applicable"

Dla etykiet klasyfikacja jedzenia nie jest wykonywana.

NOWY ENDPOINT: POBRANIE WYNIKU KLASYFIKACJI
Metoda: GET
Endpoint: photos/{photo_id}/classification

Do czego służy:
Pobiera wynik klasyfikacji dla konkretnego zdjęcia.

Przykładowa odpowiedź dla zdjęcia jedzenia:
{
  "photo_id": "550e8400-...",
  "image_type": "food",
  "classification_status": "completed",
  "predicted_class": "pizza",
  "confidence": 0.93,
  "top_predictions": [
    {"label": "pizza", "confidence": 0.93},
    {"label": "flatbread", "confidence": 0.04},
    {"label": "lasagna", "confidence": 0.02}
  ],
  "model": "ashaduzzaman/vit-finetuned-food101",
  "classified_at": "2026-03-15T12:34:56Z",
  "error_message": null
}

Przykładowa odpowiedź dla zdjęcia etykiety:
{
  "photo_id": "550e8400-...",
  "image_type": "label",
  "classification_status": "not_applicable",
  "predicted_class": null,
  "confidence": null,
  "top_predictions": [],
  "model": null,
  "classified_at": null,
  "error_message": null
}

Możliwe wartości classification_status:
- pending
- completed
- classification_failed
- not_applicable
- disabled
